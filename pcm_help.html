Quick Connect

Connessioni istantanee senza salvare un profilo:

  1. Scegli il protocollo (SSH, Telnet, SFTP, FTP, RDP, VNC)
  2. Digita la destinazione: utente@host:porta oppure solo host
  3. Premi Invio o clicca ▶ Connetti
💡 La Quick Connect chiede le credenziali al volo se non specificate. Il formato utente@host:2222 imposta utente e porta in un colpo solo.

Sidebar sessioni

  • Sessioni raggruppate per gruppo, ordinate alfabeticamente
  • Ogni riga mostra icona protocollo, nome e utente@host:porta
  • Campo ricerca live — filtra per nome, host e utente
  • Pulsanti ▼/▲ per espandere/comprimere tutti i gruppi, ↺ per ricaricare

Area schede (tab)

Ogni connessione apre una scheda. Puoi avere quante schede vuoi in parallelo.

  • Riordina le schede trascinandole con il mouse
  • Tasto destro sul tab — menu contestuale con: sposta nell'altro pannello, esporta apri-connessione.sh, esporta comandi.sh, chiudi
  • In modalità Split, le schede si distribuiscono tra i due pannelli

System Tray

Con sessioni attive, chiudere la finestra minimizza PCM nel tray invece di terminare le connessioni.

  • Doppio clic sull'icona — riapre la finestra principale
  • Tasto destro sull'icona — menu con accesso rapido alle ultime sessioni usate e opzione Esci
  • Il menu del tray mostra prima le ultime 8 sessioni usate, poi le restanti in ordine alfabetico

📋 Gestione Sessioni

Creare una nuova sessione

1
Clicca ➕ nella sidebar, oppure Ctrl+Shift+N.
2
Assegna un Nome descrittivo (es. Server produzione) e un Gruppo opzionale (es. Lavoro).
3
Scegli il Protocollo — il dialog si adatta mostrando solo i campi rilevanti.
4
Tab Connessione: Host, Porta, Utente, e le opzioni specifiche del protocollo.
5
Tab Autenticazione: password o chiave privata SSH. Puoi lasciare vuoto — PCM chiederà al momento della connessione.
6
Tab Terminale: tema colori, font, dimensione, comando avvio remoto, comando pre-connessione locale, log output.
7
Tab ⚙ Avanzate: keepalive, X11 forwarding, compressione, jump host, modalità di apertura, selettore terminale esterno.
8
Tab ⚡ Macro: definisci comandi rapidi inviabili con un clic dal pannello sessioni.
9
Tab 📝 Note: campo testo libero per annotazioni, procedure, link utili.
10
Clicca OK — la sessione viene salvata e la connessione si apre immediatamente.

Menu contestuale sessione (tasto destro)

  • ▶ Connetti — apre la connessione in una nuova scheda
  • ✏ Modifica — riapre il dialog con i dati precompilati
  • 📋 Duplica — crea una copia con suffisso "(copia)"
  • 📋 Copia comando — copia il comando shell equivalente negli appunti
  • 📄 Esporta apri-connessione.sh… — genera uno script bash standalone per aprire la connessione
  • 🌐 Verifica raggiungibilità… — testa ping, porta TCP e DNS sull'host
  • ⚡ Macro — sottomenu con le macro definite per questa sessione
  • 🗑 Elimina — richiede conferma

Importare ed esportare sessioni

Da File → Esporta sessioni salvi tutte le sessioni in un file JSON portabile. Da File → Importa sessioni carichi un file JSON precedentemente esportato. Le sessioni sono salvate in connections.json nella cartella di PCM — modificabile con qualsiasi editor.

🔌 Protocolli supportati

SSH Secure Shell

Connessione cifrata a terminale remoto. Il protocollo standard per l'amministrazione di server Linux/Unix.

  • Autenticazione — password oppure chiave privata (consigliata)
  • Jump host — server intermedio (bastion) per host non raggiungibili direttamente
  • X11 Forwarding — esegui applicazioni grafiche remote sul display locale
  • Keepalive — pacchetti periodici per mantenere viva la connessione su link instabili
  • Compressione — riduce il traffico, utile su connessioni lente
  • Comando avvio — eseguito subito dopo la connessione (es. tmux attach)
  • Pre-connessione locale — comando eseguito sul tuo PC prima di connettersi (es. attiva VPN)
  • Compatibilità legacy — opzioni per server datati (KEX, cipher, host key, MAC obsoleti)
  • Modalità apertura — terminale interno embedded oppure terminale esterno
💡 Con l'opzione "Apri browser SFTP laterale" attiva, il pannello SFTP si popola automaticamente dopo la connessione.

SFTP SSH File Transfer Protocol

Trasferimento file sicuro su canale SSH. Non è FTP — usa una connessione SSH cifrata.

  • Stessi parametri di SSH (host, porta, utente, chiave/password)
  • Browser grafico integrato stile WinSCP: navigazione, upload, download, rinomina, eliminazione
  • Visualizzazione permessi, dimensioni, date di modifica
  • Modalità apertura — browser interno, terminale SFTP CLI, file manager di sistema

FTP / FTPS File Transfer Protocol

Trasferimento file non cifrato (FTP) o con TLS esplicito (FTPS). Preferisci SFTP quando disponibile.

  • FTPS — TLS esplicito (AUTH TLS): cifra credenziali e dati
  • Modalità passiva (PASV) — consigliata dietro NAT o firewall
  • Browser grafico integrato con upload, download, navigazione
  • Modalità apertura — browser interno, terminale lftp, file manager di sistema
⚠️ FTP plain trasmette credenziali e dati in chiaro. Usa FTPS o SFTP quando possibile.

RDP Remote Desktop Protocol

Desktop remoto Windows — mostra l'intero desktop della macchina remota.

  • Client RDP — xfreerdp3 (raccomandato), xfreerdp, rdesktop
  • Dominio / Workgroup — inserisci il dominio Active Directory o il workgroup (es. AZIENDA, WORKGROUP); lascia vuoto per macchine standalone
  • Modalità apertura — Finestra esterna (default, xfreerdp in finestra separata) oppure Pannello interno (desktop RDP incorporato nella scheda PCM, richiede X11 o XWayland)
  • Schermo intero — Ctrl+Alt+Invio per uscire (solo finestra esterna)
  • Condivisione clipboard e cartelle locali
ℹ️ Il combo Client RDP mostra solo i client installati. rdesktop è il client raccomandato per il pannello interno (usa -X per embedding nativo diretto, connessione rapida senza Kerberos). xfreerdp3 supporta il pannello interno tramite xdotool ma richiede più tempo. Con xfreerdp v2 il pannello interno è disabilitato. Il campo Autenticazione permette di scegliere NTLM (veloce, senza timeout Kerberos) o Kerberos+NTLM (standard AD).

VNC Virtual Network Computing

Desktop remoto multipiattaforma (Linux, Windows, macOS) tramite protocollo RFB.

  • VNC integrato in scheda — usa noVNC per mostrare il desktop direttamente in una scheda PCM (richiede PyQt6-WebEngine)
  • Client esterno — vncviewer, realvnc-viewer, tigervnc, remmina, krdc
  • Porta default 5900; display :1 = porta 5901, :2 = 5902…
  • Password VNC separata dalla password di sistema

Telnet

Terminale legacy non cifrato. Solo per reti locali sicure o dispositivi che non supportano SSH.

🚫 Non usare Telnet su reti pubbliche — le credenziali sono visibili in chiaro sul traffico di rete.

Mosh Mobile Shell

Alternativa a SSH per connessioni instabili, roaming e alta latenza.

  • Sopravvive a disconnessioni e cambi di rete (WiFi → 4G)
  • Richiede mosh installato sia sul client che sul server
  • Usa UDP per il traffico dati, SSH solo per l'autenticazione iniziale

Seriale Console seriale

Connessione diretta a dispositivi tramite porta RS-232: router, switch, server, Raspberry Pi, Arduino.

  • Dispositivo — es. /dev/ttyUSB0, /dev/ttyACM0, /dev/ttyS0
  • Baud rate — valori comuni: 9600, 115200 (controlla la documentazione del dispositivo)
  • Se il dispositivo non risponde: sudo usermod -a -G dialout $USER (poi rilogin)
💡 Client e terminali: i combo Client RDP, Client VNC e Terminale esterno mostrano solo i programmi effettivamente installati nel sistema. Se installi un nuovo client dopo aver aperto PCM, riapri il dialog sessione per vederlo comparire. Se un profilo salvato usa un client non più installato, il combo lo mostra con il suffisso ⚠ non trovato.

🔑 Autenticazione

Password

Puoi salvare la password nel profilo o lasciarla vuota — PCM la chiederà con un dialog dedicato a ogni connessione.

⚠️ Le password salvate in connections.json sono in chiaro. Per macchine di produzione o condivise, preferisci le chiavi SSH.

Chiave privata SSH (consigliata)

1
Vai al tab Autenticazione nel dialog sessione. Nella sezione "Gestione chiavi SSH" puoi vedere le chiavi già presenti in ~/.ssh/.
2
Genera nuova chiave: scegli il tipo (ed25519 consigliato), inserisci un commento, clicca ⚙ Genera. La coppia viene salvata in ~/.ssh/.
3
Copia sul server: clicca 📤 Copia chiave pubblica sul server — PCM lancia ssh-copy-id in un xterm visibile dove puoi digitare la password del server (una sola volta).
4
Seleziona la chiave dal combo "Chiavi in ~/.ssh:" — il campo "Chiave privata" si compila automaticamente.
💡 Per non digitare la passphrase ad ogni connessione, usa ssh-agent:
eval $(ssh-agent) && ssh-add ~/.ssh/id_ed25519

Jump Host (Bastion Host)

Per raggiungere server non accessibili direttamente (DMZ, reti private):

  • Tab Autenticazione → sezione Jump Host (Bastion)
  • Jump host — indirizzo del server intermedio
  • Utente jump — utente sul bastion (può differire dall'utente finale)
  • Porta jump — porta SSH del bastion (default 22)

Il risultato è equivalente a ssh -J utente@bastion:porta utente@host-finale, gestito in modo trasparente.

ℹ️ Il Jump Host è disponibile per SSH, SFTP e Mosh.

📥 Import da altre applicazioni

Da Strumenti → Importa da applicazione esterna importi le connessioni da Remmina o Remote Desktop Manager senza reinserirle manualmente.

Importa da Remmina

  1. Vai su Strumenti → Importa da applicazione esterna → 🖥 Remmina…
  2. PCM legge la cartella ~/.local/share/remmina/ oppure puoi selezionare una cartella diversa
  3. Verificata l'anteprima (protocollo, host, numero connessioni), conferma l'importazione

Protocolli importati: SSH, SFTP, RDP, VNC, Telnet. I gruppi Remmina diventano gruppi PCM.

Importa da Remote Desktop Manager (XML o JSON)

  1. In RDM: File → Esporta → scegli formato XML (.rdm) o JSON
  2. In PCM: Strumenti → Importa → Remote Desktop Manager (XML)… oppure (JSON)…
  3. Seleziona il file, verifica l'anteprima, conferma
ℹ️ Le password non vengono importate — RDM le cifra con una chiave legata all'account. PCM chiederà le credenziali alla prima connessione.

Log output del terminale

Abilitando "Registra output su file" nel tab Terminale del profilo, xterm salva automaticamente tutto l'output in un file pcm_YYYYMMDD_HHMMSS.log nella cartella configurata.

📁 Browser SFTP e FTP

Browser SFTP integrato

Si apre automaticamente dopo una connessione SSH (se l'opzione è attiva) oppure aprendo una sessione con protocollo SFTP dedicato. Appare nel tab SFTP della sidebar sinistra.

  • Navigazione cartelle con doppio clic
  • Upload, download, creazione cartelle, rinomina, eliminazione
  • Visualizzazione permessi, dimensioni e date di modifica
  • Cronologia di navigazione (avanti/indietro)

Browser FTP/FTPS integrato

Stesso comportamento del browser SFTP, ma usa FTP o FTPS:

  • FTP plain (porta 21) o FTPS con TLS esplicito
  • Modalità passiva (PASV) per NAT/firewall

🔀 Gestione Tunnel SSH

Da Toolbar → Tunnel (oppure Strumenti → Gestore Tunnel SSH) gestisci tutti i tunnel SSH da un'unica interfaccia grafica dedicata.

ℹ️ I tunnel SSH si gestiscono esclusivamente dal Gestore Tunnel — non esiste più il protocollo "SSH Tunnel" nella lista sessioni. Questo evita duplicazioni e permette di avviare, fermare e monitorare tutti i tunnel da un solo posto.

Creare un tunnel SOCKS (proxy)

1
Apri il Gestore Tunnel da Toolbar → Tunnel oppure Strumenti → Gestore Tunnel SSH
2
Clicca + Aggiungi, scegli tipo Proxy SOCKS (-D), inserisci SSH Host e porta locale (es. 1080)
3
Clicca ▶ Avvia — il tunnel rimane attivo in background anche dopo aver chiuso il dialog
4
Configura il browser: Proxy → SOCKS5 → 127.0.0.1:1080

Tunnel locale (port forward)

Esempio: accedere a un database MySQL remoto come se fosse locale:

Tipo:          Locale (-L)
Host SSH:      server.example.com   # server con accesso al DB
Porta locale:  13306                # porta libera sul tuo PC
Host remoto:   db-interno           # hostname del DB visto dal server SSH
Porta remota:  3306                 # porta MySQL standard

# Poi connettiti con:
mysql -h 127.0.0.1 -P 13306 -u root

⚙ Funzioni avanzate

Modalità Split

Dividi l'area di lavoro per vedere più sessioni contemporaneamente:

  • Vista singola Ctrl+Alt+1
  • Split verticale Ctrl+Alt+2 — due pannelli affiancati
  • Split orizzontale Ctrl+Alt+3 — due pannelli sovrapposti

Trascina le schede tra i pannelli con il mouse, oppure usa tasto destro sul tab → "Sposta nell'altro pannello".

Multi-exec — Invia comandi a più sessioni

Da Strumenti → Multi-exec (Ctrl+Shift+M):

  • Vengono mostrate tutte le sessioni terminale attualmente attive
  • Seleziona le sessioni destinatarie
  • Scrivi il comando e scegli se inviare automaticamente Invio
  • Clicca ⚡ Invia
💡 Esempi d'uso: df -h su tutti i server, sudo systemctl restart nginx su un cluster, apt upgrade -y su tutto il parco macchine.

Variabili globali

Da Strumenti → Variabili globali (Ctrl+Shift+V). Definisci coppie NOME = valore e usale con la sintassi {NOME} nei comandi delle sessioni:

# Variabile globale:
DEPLOY_PATH = /var/www/html/app

# Campo "Comando avvio" della sessione:
cd {DEPLOY_PATH} && git pull && npm run build

Modalità protetta

Da Strumenti → Modalità protetta (Ctrl+Shift+P). Quando attiva, le password visibili nell'interfaccia vengono sostituite con ****. Utile durante screen sharing o presentazioni. Non modifica i dati salvati.

Server FTP locale

Da Strumenti → Server FTP locale (Ctrl+Shift+F). Avvia un server FTP sulla tua macchina per condividere file con altri dispositivi o server remoti. Richiede pyftpdlib.

Verifica dipendenze

Da Strumenti → Verifica dipendenze — controlla quali strumenti esterni sono installati. Utile per diagnosticare perché un protocollo non funziona.

✦ Nuove funzioni AGGIORNAMENTO

Le seguenti funzionalità sono state aggiunte nelle ultime versioni di PCM.

🔐 Cifratura credenziali

PCM può proteggere utenti e password salvati in connections.json con cifratura AES-256 (Fernet). La chiave viene derivata dalla password master con PBKDF2-HMAC-SHA256 (480.000 iterazioni) e non viene mai scritta su disco.

Primo avvio

Al primo avvio assoluto, se connections.json non esiste ancora, PCM chiede se abilitare la protezione delle credenziali:

  • Spunta "Sì, voglio cifrare le credenziali" e inserisci una password master (minimo 6 caratteri)
  • Oppure scegli "No, lascia in chiaro" per usare PCM senza protezione

Avvii successivi con cifratura attiva

PCM mostra un dialog di sblocco prima di aprire la finestra principale. Hai 3 tentativi; se li esaurisci devi riavviare l'applicazione. Il pulsante Esci chiude PCM senza sbloccare.

Gestione da menu

Da Strumenti → Imposta password globale… puoi:

  • Abilitare la cifratura — se non era attiva; tutti i profili vengono immediatamente cifrati
  • Cambiare la password — inserisci quella vecchia e quella nuova; tutti i profili vengono ricifrati
  • Disabilitare la cifratura — inserisci la password attuale; le credenziali tornano in chiaro nel JSON

Cosa viene cifrato

I campi user e password di ogni profilo in connections.json. I valori cifrati hanno il prefisso ENC: seguito dal token Fernet base64. Tutti gli altri campi (host, porte, opzioni) restano in chiaro.

⚠️ Se dimentichi la password master non è possibile recuperare le credenziali. Tieni una copia sicura della password. Per resettare: elimina connections.json e la sezione crypto da pcm_settings.json.
RDP — Pannello interno e Dominio

Le sessioni RDP hanno ora due nuove opzioni: il campo Dominio per ambienti Active Directory, e la scelta della modalità di apertura.

Campo Dominio

Nel dialog sessione RDP, tab Connessione, sezione Opzioni RDP:

  • Inserisci il dominio AD (es. AZIENDA) o il workgroup (es. WORKGROUP)
  • Viene passato come /d:DOMINIO a xfreerdp o -d DOMINIO a rdesktop
  • Lascia vuoto per macchine standalone o account locali

Modalità di apertura

ModalitàComportamentoRequisiti
Finestra esterna (default) xfreerdp si apre in una finestra indipendente, fuori da PCM Nessuno
Pannello interno Il desktop RDP appare in una scheda PCM, integrato come le sessioni SSH X11 o XWayland (DISPLAY deve essere impostato). Su Wayland puro senza XWayland non è supportato.
ℹ️ Il pannello interno usa /parent-window:<WID> di xfreerdp per agganciarsi al widget Qt. Su sessioni Wayland con XWayland attivo (il caso tipico su GNOME e KDE) funziona normalmente — PCM forza DISPLAY nell'ambiente del processo.

Wake-on-LAN

PCM può svegliare una macchina spenta prima di connettersi, inviando un magic packet UDP all'indirizzo MAC configurato. La funzione è disponibile per SSH, Mosh, RDP, VNC e Telnet.

Configurazione Wake-on-LAN

Nel dialog di modifica sessione, tab Connessione, sezione Wake-on-LAN:

  • Attiva WoL — spunta per abilitare l'invio automatico del magic packet prima di ogni connessione
  • Indirizzo MAC — indirizzo MAC fisico della scheda di rete della macchina remota (formato AA:BB:CC:DD:EE:FF)
  • Attesa risposta — secondi massimi di attesa prima che PCM tenti la connessione (default 20 s)

Come funziona

1
All'apertura della sessione, PCM costruisce il magic packet (6 byte FF + 16 ripetizioni del MAC) e lo invia in broadcast UDP sulla porta 9.
2
Appare una finestra di progresso che mostra il contatore in tempo reale mentre PCM esegue ping ripetuti verso l'host.
3
Appena l'host risponde al ping, la finestra si chiude e la connessione parte automaticamente.
4
Se il timeout scade senza risposta, PCM chiede se tentare comunque la connessione o annullare.
⚠️ Wake-on-LAN funziona solo sulla rete locale o tramite broadcast diretto. Per host su reti diverse potrebbe richiedere configurazione di router o directed broadcast. Assicurati che la scheda di rete e il BIOS/UEFI della macchina remota abbiano WoL abilitato.
Verifica raggiungibilità

Testa la connettività verso un host prima (o dopo) di tentare la connessione, con tre controlli distinti in sequenza.

Come usarla

Tasto destro su qualsiasi sessione nel pannello → 🌐 Verifica raggiungibilità…

Si apre un dialog con log in stile terminale che esegue automaticamente:

TestCosa verificaRisultato
1. DNS Risolve il nome host in indirizzo IP ✅ IP risolto oppure ❌ nome non trovato
2. Ping Invia 3 pacchetti ICMP, mostra RTT e perdita ✅ Ping OK oppure ⚠ ICMP bloccato (non significa che l'host sia spento)
3. Porta TCP Apre una connessione TCP sulla porta del protocollo ✅ Porta aperta oppure ❌ timeout / rifiutata
ℹ️ Il ping bloccato (ICMP filtrato dal firewall) non significa che il servizio sia irraggiungibile — il test sulla porta TCP è quello definitivo. Un host può rispondere sulla porta SSH pur non rispondendo al ping.
Macro per sessione

Comandi rapidi collegati a una specifica sessione, inviabili al terminale con un singolo clic dal pannello sessioni.

Definire le macro

Nel dialog di creazione/modifica sessione, tab ⚡ Macro:

  • Nome — etichetta che appare nel menu (es. Stato servizi)
  • Comando — il comando da inviare al terminale (es. systemctl status nginx)

Usa i pulsanti ➕ Aggiungi, ✏ Aggiorna, 🗑 Elimina, ▲ / ▼ per gestire l'ordine.

Usare le macro

Tasto destro sulla sessione nel pannello → ⚡ Macro → seleziona il comando.

Il comando viene inviato istantaneamente al terminale attivo della sessione tramite clipboard X11 (nessun ritardo di digitazione). Se la sessione non è aperta, PCM avvisa.

💡 Le macro sono visibili anche nel tooltip della sessione nel pannello (passaci sopra con il mouse). Usa nomi descrittivi come Tail log nginx, Riavvia Apache, Spazio disco per trovare subito quello che cerchi.

Esempi di macro utili

  • df -h — spazio disco
  • tail -f /var/log/syslog — log di sistema live
  • sudo systemctl restart nginx — riavvio servizio
  • uptime && free -h — stato sistema
  • docker ps — container attivi

Differenza con il Comando avvio

Il Comando avvio (tab Terminale) viene eseguito automaticamente alla connessione. Le macro vengono eseguite manualmente su richiesta, in qualsiasi momento durante la sessione.

Comando pre-connessione locale

Esegui un comando shell sul tuo PC prima che PCM apra la connessione remota. Ideale per attivare VPN, montare volumi o qualsiasi operazione locale preparatoria.

Configurazione

Nel dialog di modifica sessione, tab Terminale:

  • Cmd locale pre-connessione — comando bash da eseguire prima di connettersi
  • Timeout pre-cmd — secondi massimi di attesa (0 = nessun timeout, default 15 s)

Come funziona

Il comando viene wrappato con il comando di connessione in un unico processo bash:

bash -c 'pre-cmd && echo "Pre-connessione OK. Connetto..." && ssh utente@host'

Il terminale mostra l'output del pre-comando. Se il pre-comando esce con codice diverso da zero (errore), la connessione non parte e viene mostrato il messaggio di errore.

Esempi pratici

  • wg-quick up vpn0 — attiva VPN WireGuard
  • openfortivpn --config=/etc/vpn.conf — attiva FortiVPN
  • sudo openvpn --config ~/vpn.ovpn & — avvia OpenVPN
  • sshuttle -r gateway 10.0.0.0/8 & — tunnel rete

Nota sul timeout

Imposta il timeout a un valore ragionevole per il tuo pre-comando. Una VPN che impiega 10 s ad attivarsi richiede un timeout di almeno 15–20 s. Con 0 il pre-comando può girare indefinitamente (utile per comandi in background).

Esporta apri-connessione.sh

Genera uno script bash standalone che apre la connessione senza avere PCM aperto. Utile per condividere o per uso da terminale.

Come usarla

Tasto destro sulla sessione nel pannello → 📄 Esporta apri-connessione.sh…

Si apre un dialog con:

  • Anteprima dello script generato (modificabile prima di salvare)
  • 💾 Salva .sh… — salva con chmod 755 automatico, pronto all'uso
  • 📋 Copia negli appunti — per incollare direttamente in un terminale

Lo script include, se configurati: blocco pre-cmd locale, blocco Wake-on-LAN (con fallback Python puro se wakeonlan non è installato), e il comando di connessione.

#!/usr/bin/env bash
# PCM — Sessione esportata: Server produzione
# Protocollo: SSH   Host: 192.168.1.10

set -e

# Pre-connessione locale
wg-quick up vpn0

# Wake-on-LAN
wakeonlan "AA:BB:CC:DD:EE:FF" 2>/dev/null || true
echo "WoL inviato, attendo 20s..."
sleep 20

# Connessione
ssh -p 22 -o StrictHostKeyChecking=no [email protected]
Esporta comandi.sh

Genera un replay script con tutti i comandi eseguiti durante la sessione. Il replay si connette via SSH e ri-esegue i comandi in sequenza — ideale per automatizzare procedure ripetitive o documentare operazioni di manutenzione.

Come usarla

Tasto destro sul tab della sessione attiva (non sul pannello sessioni) → 📄 Esporta comandi.sh…

⚠️ Questa funzione è accessibile dal tab del terminale, non dal pannello sessioni. Il pannello sessioni ha "Esporta apri-connessione.sh" (script di connessione), il tab ha "Esporta comandi.sh" (replay dei comandi).

Log abilitato vs non abilitato

PCM distingue due scenari in base alla configurazione del log di sessione:

🔴 Log non abilitato

Appare un avviso con le istruzioni per abilitare il log. È comunque possibile esportare i comandi inviati tramite macro e multi-exec con il pulsante "Esporta solo comandi PCM".

🟢 Log abilitato

PCM parsa il file di log xterm, estrae i comandi digitati riconoscendo le righe che seguono un prompt bash/zsh, e li unisce ai comandi tracciati internamente.

Abilitare il log per usare l'export completo

1
Tasto destro sulla sessione → ✏ Modifica
2
Tab Terminale → spunta "Registra output su file"
3
Imposta la Cartella log (default: /tmp/pcm_logs)
4
Chiudi e riapri la sessione — da questo momento i comandi vengono registrati
5
Esegui le operazioni che vuoi documentare, poi usa "Esporta comandi.sh"

Il dialog di export

Il dialog mostra una tabella con tutti i comandi trovati, ciascuno con:

ColonnaSignificato
TimestampOrario di esecuzione (ISO 8601)
Sorgentelog_xterm (digitati), macro, multi_exec, startup_cmd
ComandoTesto del comando — editabile con doppio clic

Puoi modificare i comandi con doppio clic, riordinarli con ▲/▼, eliminare quelli indesiderati. L'anteprima dello script si aggiorna in tempo reale.

Script generato

Per sessioni SSH, lo script usa un heredoc per eseguire tutti i comandi in una sola connessione:

#!/usr/bin/env bash
# comandi.sh — Server produzione
# SSH [email protected]:22
# 5 comandi

set -e

ssh -p 22 -o StrictHostKeyChecking=no [email protected] << 'PCMREPLAY'
df -h
systemctl status nginx
tail -n 50 /var/log/nginx/error.log
docker ps
journalctl -n 100 --no-pager
exit
PCMREPLAY
💡 Lo script generato è completamente editabile sia nel dialog di anteprima che dopo il salvataggio. Aggiungere commenti, condizioni o loop è un'operazione normale prima di usarlo in produzione.

🛠 Impostazioni globali

Da Toolbar → Impostazioni (o File → Impostazioni) — quattro tab:

Tab Generale

  • Home directory — cartella di partenza per i file dialog
  • Editor di testo — usato per aprire file di configurazione (nano, vim, gedit, code…)
  • Conferma alla chiusura — chiede cosa fare quando ci sono sessioni attive

Tab Terminale

  • Tema predefinito — schema colori per tutte le nuove sessioni (sovrascrivibile per singola sessione)
  • Font predefinito e Dimensione font
  • Righe scrollback — quante righe mantenere in memoria (100–100.000)
  • Incolla con tasto destro
  • Conferma chiusura tab — avvisa prima di chiudere un tab con processo attivo
  • Avviso paste multiriga — evita esecuzioni accidentali
  • Registra output su file + Cartella log — globale per tutte le sessioni che non hanno un log specifico

Tab SSH

  • Keepalive interval — secondi tra i pacchetti keepalive (0 = disabilitato)
  • StrictHostKeyChecking — verifica impronta del server SSH
  • Apri browser SFTP automaticamente — popola il tab SFTP della sidebar a ogni connessione SSH

Tab Scorciatoie

Personalizza tutte le scorciatoie. Formato: Ctrl+Shift+X, F5, ecc. Le modifiche hanno effetto al prossimo avvio.

❓ Domande frequenti

Le icone non appaiono — vedo solo pallini colorati

Le icone PNG devono essere presenti nella cartella icons/ accanto a PCM.py. Verifica i file .png. Se hai solo .svg, esegui bash convert_icons.sh dalla cartella di PCM.

La connessione SSH fallisce con "Host key verification failed"

Il server SSH ha cambiato la sua impronta. Rimuovi la vecchia voce:

ssh-keygen -R hostname

Oppure disabilita StrictHostKeyChecking nel profilo (tab Avanzate) o nelle impostazioni globali.

xterm non si apre / errore "cannot open display"

xterm richiede un display X11. Verifica di essere in una sessione grafica. Controlla che xterm sia installato: which xterm.

RDP si apre ma lo schermo è nero

Prova a cambiare client da xfreerdp3 a xfreerdp nel profilo. Su alcune versioni di Windows Server potrebbe servire aggiungere /sec:rdp tra le opzioni avanzate.

Il browser SFTP non si connette

Il browser SFTP usa paramiko. Verifica l'installazione:
.venv/bin/python -c "import paramiko"
Se manca: uv pip install --python .venv/bin/python paramiko

Le macro sono lente / arrivano con ritardo

PCM invia le macro tramite clipboard X11 (xclip o xsel) invece di simulare la tastiera tasto per tasto — è istantaneo indipendentemente dalla lunghezza del comando. Se riscontri ritardi, verifica che xclip sia installato: which xclip. Se manca: sudo apt install xclip.

"Esporta comandi.sh" non trova comandi

I comandi digitati direttamente in xterm vengono catturati solo attraverso il log xterm. Assicurati che nel profilo della sessione sia attivo "Registra output su file" e che la sessione sia stata riconnessa dopo aver abilitato il log. I comandi inviati tramite macro e multi-exec sono sempre tracciati indipendentemente dal log.

Wake-on-LAN non sveglia la macchina

Verifica: (1) WoL abilitato nel BIOS/UEFI della macchina remota; (2) WoL abilitato sulla scheda di rete — ethtool eth0 | grep Wake; (3) la macchina è sulla stessa rete locale o il router permette il broadcast diretto; (4) l'indirizzo MAC è corretto.

Come faccio a connettere a un server con porta SSH non standard?

Specifica la porta nel campo "Porta" del dialog sessione oppure usa il formato utente@host:2222 direttamente nella barra Quick Connect.

Posso avere PCM su più macchine con le stesse sessioni?

Sì — il file connections.json è portabile. Sincronizzalo con Dropbox, Nextcloud, Git o copialo manualmente. Usa File → Esporta sessioni per backup e trasferimenti.

I tunnel SSH si chiudono quando chiudo PCM?

Se minimizzi nel system tray i tunnel rimangono attivi. Se esci definitivamente, tutti i processi SSH vengono terminati. Per tunnel permanenti, usa un systemd user service o screen/tmux sul server.

Dove sono salvate le impostazioni?

  • connections.json — tutte le sessioni (in chiaro, JSON modificabile)
  • pcm_settings.json — impostazioni globali, scorciatoie, sessioni recenti
  • /tmp/pcm_logs/ — log dei terminali (se abilitato, percorso configurabile)
